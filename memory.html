how it scales

memory is the state. agents are stateless.

the multi-tenant pattern: per-user file namespace + ephemeral agent runtime + thin metadata index for routing. each request spins up a fresh agent that loads exactly the requesting user's memory tree, processes the turn, writes back, exits. memory persists. agents don't.

storage isolation

per-user namespace = filesystem-trivial multi-tenancy. user A's agent literally cannot read user B's files. no application-level access control bugs possible.

stateless compute

1000 concurrent users does NOT mean 1000 long-running agent processes. each request spins up an ephemeral agent: load memory, do work, exit. peak concurrency is realistic (10–50), not user count.

no vector tax

file-based + grep means zero embedding-API cost, zero vector-DB hosting cost. competitors pay $0.10–0.30 per active user just in infra before LLM. we don't.
